Filter chain · _fetch_one() + _quality_flags()

Every headline passes through this chain in order. First match that applies wins. Each step can exclude, demote, or flag a story.

STEP 1 — HARD EXCLUDE
Satire · Press release · Junk results
SATIRE_DOMAINS · _PRESS_RELEASE_RE · _RESULTS_RE
Story dropped entirely. Never shown anywhere. Logged as excluded in feed worker.
STEP 2 — FORCE NEGATIVE
Known bad patterns regardless of VADER score
_FORCE_NEGATIVE_RE → compound = min(compound, −0.06)
Catches: war crimes, massacre, genocide, ethnic cleansing, airstrike, civilian killed, children killed, voting rights suppression, financial crime verbs (embezzle, defraud etc). VADER cannot be trusted on these — override.
STEP 3 — THRESHOLD GATE
Positive threshold check
POSITIVE_THRESHOLD = 0.15 · NEGATIVE_THRESHOLD = −0.05
compound ≥ 0.15 → pro candidate. compound ≤ −0.05 → con. Between → neutral → flagged. Raised from 0.05 on 16 May 2026 to reduce false positives.
STEP 4 — WAR & CONFLICT GATE
Topic-level override for conflict stories
if _get_topic(title) == 'War & Conflict' and compound < 0.5: sarcasm_risk = True
War headlines almost never belong in pro feed. Requires compound ≥ 0.5 (strongly positive — e.g. "War ends, peace deal signed") to pass. Everything else → flagged.
STEP 5 — NEGATIVE CONTEXT WORD CHECK
Word-level negative context detection
_NEGATIVE_CONTEXT (80+ words) → sarcasm_risk = True
If compound ≥ 0.15 AND any word in title matches the set → sarcasm_risk. Covers: crisis, war, kill, disaster, fraud, corruption, survivor, victim, refugee, ceasefire, terror, suicide, overdose, trafficking, civilian, airstrike, and 60+ more.
STEP 6 — QUALITY FLAGS
Scare quotes · Clickbait · Opinion · Framing risk
_SCARE_QUOTE_RE · _CLICKBAIT_RE · _OPINION_STARTERS · _FRAMING_RISK_RE
Scare quotes or question marks → sarcasm_risk. Clickbait openers → clickbait flag. Opinion starters ("Why ", "Analysis: ") → opinion flag. Geographic framing errors → framing_risk flag.
STEP 7 — ROUTING
Final classification decision
pro + (sarcasm_risk OR framing_risk) → flagged | pro → pro | con → con | neutral → flagged
Only clean pro stories with no risk flags reach the pro feed. Everything with any flag goes to Held. Con goes to filtered. This is the last gate before display.

External services · dependencies + failure modes
RSS · 32 sources
News feeds
feedparser parses each source. 25s thread timeout. Individual source failures tracked in _source_errors{}. Consecutive errors shown in sources panel.

If fails: that source marked offline, rest of feed unaffected.
ResilientPer-source fallback
Sentiment · Local
VADER
vaderSentiment — installed Python package. Runs entirely in-process. No external call. Cannot fail independently.

If fails: entire pipeline breaks — app crashes.
LocalCritical dependency
Database · Cloud
Neon PostgreSQL
DATABASE_URL env var. ThreadedConnectionPool(2–10) — no fresh TCP per request. _db_conn() is a contextmanager yielding pooled connections. _log_pageview() runs in a daemon thread — never blocks the page load. All DB functions wrapped in try/except — graceful degradation.

If fails: feed still works. Dispatch/ads/analytics degrade gracefully.
NeonGraceful degradation
Payment · Cloud
Stripe
STRIPE_SECRET_KEY + STRIPE_WEBHOOK_SECRET env vars. Used only for Shoutout feature (£1.50). Belt-and-braces: both redirect confirm AND webhook mark payment.

If fails: Shoutout disabled, rest of site unaffected.
RevenueIsolated
Deployment · Cloud
Railway
Auto-deploys on push to main. Procfile: web: gunicorn app:app. Hobby pack — 5 slots, 3 used. PORT env var. 512MB RAM limit — VADER + feedparser are the heavy users.

SELECT * FROM ads WHERE slot=%s AND active=TRUE ORDER BY created_at ASC
Returns list of paid ads. If list empty → returns [self_promo fallback].

⚡ Injection points · where to add features

Each marked point is a clean seam in the architecture. Plug in without touching unrelated code.

app.py → _classify() or after _quality_flags()
ML sentiment model
Replace or augment VADER with a fine-tuned news-domain model (HuggingFace distilbert-base-uncased-finetuned-sst-2). VADER is fast but news-naive. A trained model could reduce false positives further. Injection: add a second compound score, average or take the more conservative.
app.py → after _record_score() in _fetch()
Push notification alert
When score drops below 30, send a web push to all subscribed PWA users. Needs: a push subscription table, Web Push library (pywebpush), VAPID keys. The service worker is already registered — only backend work needed.
app.py → new route + subscribers table
Email digest sender
The subscriber list is building. Add a POST /api/admin/send-digest route that: pulls top 5 pro stories from _state, formats them as an HTML email, calls Brevo or ConvertKit API to send to confirmed subscribers. Run daily via a cron job or Railway scheduled task.
app.py → new route /api/v1/score
ProCon API product
The score_history table is now populating every 30 minutes. A paid API that exposes historical score + topic breakdown is a B2B product. Add: API key table, rate limiting middleware, a /api/v1/score endpoint that returns {score, pro_count, con_count, topics, timestamp} per day. Charge £99-500/month per subscriber.
FEEDS list in app.py
New RSS sources (Finance, Science, etc.)
Add any RSS-enabled source as a tuple: (name, rss_url, bias_label, website). Immediate effect on next fetch cycle. Candidates: Reuters Business (feeds.reuters.com/reuters/businessNews), MarketWatch (feeds.marketwatch.com/marketwatch/topstories/), The Economist Finance (economist.com/finance-and-economics/rss.xml).
_SELF_PROMOS dict + index.html masthead
Leaderboard ad unit
Add a 'leaderboard' slot between the masthead and the lede. 728×90 style. The highest-value digital ad position (above-the-fold, below brand). Needs: new slot in _SELF_PROMOS, new ad_slot() call in template, CSS for the unit. Estimated value: 2-3× CPM of strip ads.
templates/index.html → premium tier gate
£2.99/month premium tier
Stripe subscription (not one-time). Add: stripe_subscription_id to a new subscriptions table, a /subscribe/premium route, a session flag for premium users. Premium unlocks: no ads (check premium flag in ad_slot macro), push alerts, archive access. The Stripe infrastructure is already wired.
app.py → _get_regional_stories() or new route
Topic trend graph
The story_clicks table records topic per click. Add an endpoint that aggregates topic click counts over 7 days and returns a ranked list. Render as a small bar chart in the sidebar or below sources. Shows which topics readers are actually reading — editorial intelligence.
Railway — 2 unused slots
Standalone services
Slots 4 and 5 of the Hobby pack are empty. Candidates: BeaconLogger (ready to deploy — silent asset-ping system), or a dedicated ProCon API microservice (separate from MeDea, serving the B2B API product cleanly).
